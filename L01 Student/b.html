Web Development Process

Web Development Process made by Sean Yong

1. Research and Analysis

- Identify the objective, problems, and opportunities of the project

- Determine the duration/schedule of the project and its feature

- Determin the necessary manpower needed and their skill-set

- Identify the projects stakeholders and gather the requirements to make a detailed analysis

- Although it may change later, define the initial project scope


2. Design

- Create moodboard and identify the fonts used to create consistency

- Create sitemap to show the features of the projecy

- Create a low fidelity wireframe based on the sitemap

- Create a hig fidelity wireframe based on low fidelity wireframe


3. Development

- Create the projects content

- Code the features of the project

- Ensure each features are able to work together

- Test and evaluate the website with a 3rd-party

- Fix the problems/bugs found during the test


4. Launch

- Finalize the design and coding

- Ensure the website is able to work with other servers

- After passing the previous steps, release it to the public


5. Maintainance

- Receive reports from customer. Reports may vary from bug, glitches, etc.

- Update the website after fixing bugs, making enhancements, or upgrading the software of the website.
